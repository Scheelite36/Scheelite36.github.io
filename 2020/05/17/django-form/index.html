表单相关介绍与使用

表单的介绍

常见的表单元素：

  1. 表单标签——<form>
  2. 单行文本框/多行文本框
  3. 选择（单选、多选、下拉选择）
  4. 隐藏表单域
  5. 表单按钮
  6. 文件上传框

<form>标签：

  1. action——表单提交的URL地址
  2. method——表单请求的方式（GET/POST）
  3. enctype——请求内容的形式
    application/x-www-form-urlencoded
   multipart/form-data（用于上上传文件，如word、excel等）

文本标签：

  1. textarea——多行文本

  2. 单行文本（对应input中的type值）

    text password email url number color 时间日期等（date、month、week、time、date、datetime、datetime-local）

选择标签：

  1. 单选多选radio checkbox

  2. 下拉选择

    其他：file上传文件 button hidden隐藏表单域

在视图中获取表单值：

  1. GET请求
    request.GET.get('name', None)
  2. POST请求
    request.POST.get('name', None)

Django内置的字段

Form对象的属性和方法：
is_bound 是否已经绑定数据
is_valid() 表单是否已经通过验证
cleaned_data 访问表单验证后的数据
as_p/as_ul/as_table 渲染表单（在表单html中添加）
errors 表单验证后的错误信息
fields 表单中的字段
initial 初始化数据（指定value值）

  • 表单字段继承自django.forms.Field
  • 每个字段都具有核心参数
  • 每个字段都可以有自定义的验证逻辑
  • 每个字段都有钩子函数，方便扩充功能

字段的常用核心参数：
required 是否为必填，默认为必填
label label标签（如：输入框前的文字描述）
initial 初始化数据
widget 定制界面显示方式（如 文本框、选择框）
help_text 帮助文字
error_messages 覆盖字段引发异常后的错误显示
localize 本地化，根据用户所在地区格式进行显示
disabled 禁用表单，界面上不可操作
has_changed() 值是否发生了改变

Django内置的字段（与ORM模型中字段类似）：
与HTML表单的各种类型对应

文本/字符串
CharField 字符串输入
EmailField 邮件地址输入
URLField URL地址输入
UUIDField uuid字符串输入

数值（整数、小数）
FloadField 浮点数输入
IntegerField 整数输入
DecimalField 小数输入（更精确）

选择
ChoiceField 单选
MutipleChoiceField 多选
TypedChoiceField 高级选择（支持结果转换类型）

日期/时间
FileField 文件
FilePathField 文件路径
ImageField 图片

布尔
BooleanField True/False
NullBooleanField None/True/False

正则
RegexField 正则输入

是否可以通过DjangoORM模型来创建表单？

假设已经有模型(Model)了，但是要设计对应的表单(Form)，此时又很多公用的字段，为了方便创建表单，可以使用django的ModelForm

ORM字段 表单字段
ForeignKey ModelChoiceField
ManyToManyField ModelMultipleChoiceField
TextField CharField
PositiveIntegerField IntegerField

可以在官方文档中搜索ModelForm查看各种对应字段。

修改模型表单主要是通过Meta来修改

Meta配置选项：

  • widgets——修改展现样式（文本输入、数字输入）
  • help_texts——设置帮助文字
  • labels——设置表单输入前的文字提示
  • error_messages——设置表单的错误提示
